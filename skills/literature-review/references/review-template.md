# 文献梳理报告模板

Skill 2 的 Markdown 渲染器 `renderer.render_literature_report` 严格按下面的
结构产出。结构稳定有两个好处：
1. Skill 3（大纲）可以可靠地从「研究脉络 → 共识 → 争议 → 缺口」段落直接复用。
2. 人工审阅时定位信息更快。

## 顶层结构

```
# 文献梳理报告 · {topic}
> case_id: `{case_id}`

- 关键词：{keywords}
- 引用风格：{citation_style}
- 文献条目：{N} 篇

## 一、研究脉络
> {timeline_summary}

### {cluster_name}
{cluster_summary}

代表性文献：
- `{paper_id}`
- ...

## 二、共识 · 争议 · 研究缺口

**共识：**
- ...

**争议：**
- ...

**研究缺口：**
- ...

## 三、文献明细

| ID | 年份 | 题目 | 作者 | 核心观点 | 证据强度 | 对齐 |

## 四、规范参考文献

### GB/T 7714-2015
[1] ...

### APA 7
- ...
```

## 写作建议

- **研究脉络**：用 50-100 字概括领域演进；不要逐篇罗列，而要按"主线"叙述。
- **共识**：必须是"多数文献认同"的命题。
- **争议**：标出"哪两种观点 / 哪些证据互相矛盾"。
- **研究缺口**：每条应与 `writing_task.core_arguments` 中的至少一条对齐。
- **文献明细表**：尽量保留 3 位以内作者；超过的用「等」省略。
