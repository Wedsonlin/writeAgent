# writeAgent 案例 · 用户原始需求

## 写作背景

我正在做"高级人工智能"课程的论文写作 Agent 与 Skill 设计作业，希望写一篇
**面向学术论文写作的智能 Agent 设计与实现**的论文，把本项目的架构、Skill
划分、OpenClaw 平台适配等思路系统化地呈现出来。

## 写作要求

- **论文类型**：系统设计类（含一定综述成分）。
- **目标期刊**：CCF-B 级中文期刊，**《计算机研究与发展》**风格。
- **字数**：约 10000 字（8000-12000 字之间）。
- **写作语言**：中文。
- **核心论点**（按重要性排序）：
  1. "大脑决策 + Skill 工具调用"是当前 LLM Agent 的主流范式，并能显著降低
     学术论文写作门槛；
  2. **统一输入输出字段**（用 JSON Schema 约束）是多 Skill 协作能稳定串联
     的关键保障；
  3. **LangGraph + OpenClaw 双轨编排**架构可以同时兼顾本地开发调试与平台部署。
- **创新点 / 主要贡献**：
  - 提出 LangGraph 编排 + OpenClaw 兼容 Skill 的双轨架构；
  - 提出以 JSON Schema 作为跨 Skill 契约的统一基线；
  - 针对论文写作场景设计了层次清晰的 6 个 Skill 划分。
- **研究范围**：
  - 涉及：LLM Agent、学术写作辅助、工具编排、Skill 化扩展机制、JSON Schema 契约设计；
  - 不涉及：模型本身的预训练 / 微调；不评测商业查重服务。
