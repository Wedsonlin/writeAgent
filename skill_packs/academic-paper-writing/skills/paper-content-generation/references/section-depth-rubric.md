# Section Depth Rubric

Use this rubric when a draft section is long enough but still reads like an expanded outline.

Each substantive section must contain six visible moves:

- `problem_framed`: state the local research problem, tension, or claim the section resolves.
- `mechanism_explained`: explain how the system, method, infrastructure layer, or scholarly relationship works.
- `evidence_interpreted`: interpret cited literature or data instead of only attaching a citation marker.
- `comparison_or_tradeoff`: compare alternatives, layers, systems, periods, or tradeoffs.
- `limitation_or_boundary`: state what the evidence does not prove, where the claim is weak, or what remains open.
- `argument_return`: explicitly return to the paper's core argument or innovation point.

Do not mark a move as complete unless the section body contains corresponding prose. A list of papers, systems, or keywords is not enough.

For AI infrastructure survey writing, prefer paragraphs that connect:

- resource pressure -> system mechanism -> evidence from representative papers;
- training/inference/data/platform layers -> cross-layer coordination;
- open-source systems vs cloud platforms -> governance, cost, and reproducibility tradeoffs;
- observed trend -> limitation -> what the paper can and cannot conclude.
