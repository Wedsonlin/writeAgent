# 04 · 案例主题与执行计划

## 一、案例主题

**面向学术论文写作的智能 Agent 设计与实现**（自指性案例）

- **类型**：系统设计（含一定综述成分），`paper_type = "system"`
- **目标期刊**：CCF-B 级中文期刊《计算机研究与发展》风格
- **写作语言**：中文
- **字数**：8000-12000 字（默认按 10000 字配章节预算）
- **核心论点**：
  1. "大脑决策 + Skill 工具调用"模式可显著降低学术写作门槛
  2. 统一输入输出字段是多 Skill 协作能稳定串联的关键保障
  3. LangGraph + OpenClaw 双轨编排可兼顾本地开发与平台部署
- **创新点**：
  - LangGraph + OpenClaw 双轨架构
  - JSON Schema 作为跨 Skill 契约的统一基线
  - 针对论文写作场景设计的 6 个层次清晰的 Skill 划分
- **研究范围**：
  - 涉及：LLM Agent · 学术写作辅助 · 工具编排 · Skill 化扩展机制 · JSON Schema 契约设计
  - 不涉及：模型本身的预训练 / 微调；商业查重服务的评测
- **参考文献种子**：[`case/references/seed.bib`](../case/references/seed.bib) 共 16 篇，覆盖 2020-2025 年

## 二、为什么选这个主题

| 维度 | 解释 |
| --- | --- |
| 自指性 | 案例主题就是项目本身，写作的素材天然存在，无需额外采集 |
| 数据完整 | 项目代码、设计文档、Schema、参考文献全部可作为论文论据 |
| 覆盖三类内容 | 综述（Agent 现状）+ 系统（架构与 Skill）+ 案例（自指闭环） |
| 与课程匹配 | 选自"高级人工智能"课程作业，可作为期末答辩材料 |

## 五、与组员的衔接

### 1. 张慧晴（Skill 3 / Skill 4）

- **输入约定**：从 `state.writing_task` + `state.literature_report` 读取所有
  需要的字段；不要自行重新解析文献。
- **输出约定**：严格按 `schemas/paper_outline.schema.json` 与
  `schemas/paper_draft.schema.json` 写回 `state.json`。
- **Markdown 渲染**：使用统一的章节编号体系（点分十进制），引用以 `[n]` 形式
  内联。
- **本地联调**：跑 `python examples/run_skill1_then_skill2.py` 得到上游产物后
  即可调试 Skill 3。

### 2. 李泓铮（Skill 5 / Skill 6）

- **输入约定**：Skill 5 从 `state.draft` + `state.writing_task.target_journal.style_profile` 起步；Skill 6 从 `state.formatted_draft.normalized_draft` 起步。
- **输出约定**：见 `schemas/format_report.schema.json` 与 `schemas/polish_report.schema.json`。
- **docx 导出**：可直接使用 `python-docx`；模板放在 `assets/`，路径透过 `--template` 传入。
- **PDF 导出**（可选）：建议优先用 `pandoc`；若 OpenClaw 沙盒不便安装，可改为输出 `.docx` 并提示用户本机转换。

## 六、风险预案

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| Skill 3 / 4 字数失衡 | Skill 5 校验报警 | Skill 3 实现 `balance.py` 做 normalization |
| Skill 4 引用错位 | 学术规范性受损 | Skill 4 使用 `citation_inject.py`，且 Skill 5 做最终 sanity check |
| PDF 导出依赖问题 | Skill 5 在沙盒中失败 | 默认输出 markdown + docx；PDF 作为附加 |
| LLM 服务不可用 | 端到端流水线中断 | 全链路 Mock 兜底；阶段 PPT 准备一份 Mock 跑通的录屏 |
| 文档 / Skill 输出字段漂移 | 模块对接失败 | 改 Schema 必须先发群通知 + 同步 pydantic 模型 |
